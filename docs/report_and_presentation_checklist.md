### Discussion

The discussion section should be rewritten in the group members' own wording after the simulation results are understood. Important points to include:

- PID is stable, interpretable, and easy to tune for a known plant.
- The ML controller imitates PID behaviour because it is trained using PID-generated expert data.
- The ML controller depends on the quality and range of the training data.
- The results are similar because both controllers are based on the same control behaviour.
- PID remains more predictable outside the tested operating range.
- ML can be improved by adding more training scenarios or using reinforcement learning.

## Presentation Improvements

Recommended slide flow:

1. Title and group members
2. Project problem statement
3. Reason for selecting DC motor
4. DC motor mathematical model
5. PID controller design
6. ML neural network controller design
7. Simulation setup
8. Results graphs
9. Performance comparison table
10. Conclusion and limitations

The presentation should focus on the project story:

```text
DC motor model -> PID baseline -> ML controller training -> simulation comparison -> findings
```

Detailed code explanations should be kept short. The main focus should be the control objective, controller design, results, and interpretation.

## Common Questions To Prepare

### Why is the ML result close to PID?

The neural network was trained using PID-generated expert data, so it learns to imitate PID control behaviour.

### Why use a DC motor?

A DC motor is a common mechatronic plant, has a clear mathematical model, and is suitable for speed-control simulation.

### What is the limitation of the ML controller?

The ML controller may not generalize well to reference speeds, disturbances, or plant parameters that are very different from the training data.

### Which controller is better?

For this simulation, both controllers perform similarly. PID is more interpretable and predictable, while the ML controller demonstrates how a controller can learn useful control behaviour from data.

## Abstract

This mini project presents the design and simulation of an intelligent controller for DC motor speed control. A mathematical model of a DC motor is developed using electrical and mechanical differential equations. A classical PID controller is implemented as the baseline controller. A Machine Learning-based controller is then developed using a small neural network trained from simulated expert-control data. The performance of both controllers is evaluated using rise time, overshoot, settling time, mean squared error, control effort, and robustness under a load-torque disturbance. The results show how a neural network controller can learn useful control behaviour, while also highlighting the interpretability and reliability advantages of the PID controller.

## Introduction

Classical controllers such as P, PI, and PID controllers are widely used in control systems because they are simple, reliable, and easy to tune. However, their performance can decrease when the controlled plant is nonlinear, uncertain, or affected by disturbances. Machine Learning techniques can be used to create intelligent controllers that learn control behaviour from data and adapt to different operating conditions.

For this project, a DC motor is selected as the plant. DC motors are common in mechatronic systems, robotics, automation, and electric drives. The control objective is to make the motor speed follow a desired reference speed while rejecting disturbance caused by external load torque.

## Objectives

- To model and simulate a DC motor speed-control system.
- To design a classical PID controller for speed tracking.
- To develop a Machine Learning-based neural network controller.
- To compare both controllers using suitable control-performance metrics.
- To discuss the strengths and limitations of both approaches.

## System Selection and Modelling

The DC motor is modelled using two state variables: angular speed and armature current. The input to the system is the applied motor voltage. The disturbance is represented as an external load torque.

The electrical equation is:

```text
L di/dt + R i + K omega = V
```

The mechanical equation is:

```text
J d omega/dt + b omega = K i - T_load
```

Where:

- J is the rotor inertia.
- b is the viscous friction coefficient.
- K is the motor torque and back-emf constant.
- R is the armature resistance.
- L is the armature inductance.
- V is the input voltage.
- i is the armature current.
- omega is the motor angular speed.
- T_load is the disturbance load torque.

## PID Controller Design

The PID controller calculates the voltage command using the speed error, the accumulated error, and the rate of change of error. The general PID control law is:

```text
V = Kp e + Ki integral(e) + Kd de/dt
```

The PID controller is used as the baseline because it is a standard classical controller and is suitable for comparison with the ML controller.

## Machine Learning Controller Design

The ML controller is implemented as a feedforward neural network. The neural network receives control-related features such as reference speed, actual speed, speed error, integral error, and derivative error. It outputs the motor voltage command.

The model is trained using supervised learning. Training data is generated by simulating the DC motor with a PID expert controller under different reference speeds and disturbance conditions. The neural network learns to imitate the expert controller's control action from these examples.

## Simulation Setup

The simulation is implemented in Python using NumPy. The motor equations are solved using a fixed-step numerical integration method. Both controllers are tested using the same motor parameters, reference speed, voltage limit, and disturbance profile. A load-torque disturbance is applied during the simulation to compare disturbance rejection.

The generated output files are stored in the `outputs` folder:

- `speed_response.svg`
- `control_signal.svg`
- `training_loss.svg`
- `metrics.csv`
- `results_summary.md`

Expected discussion points:

- The PID controller is easier to understand and tune.

- PID is usually more predictable outside the tested operating range.
- ML can be improved by adding more training scenarios or using reinforcement learning.

## Discussion

The PID controller provides a strong classical baseline because its control action is directly based on the speed error. Its behaviour is interpretable and predictable. However, PID tuning can become difficult for nonlinear or time-varying systems.

The neural network controller demonstrates an intelligent-control approach. Instead of explicitly programming the control equation, the controller learns the relationship between the motor state and the required voltage command. This is useful when the control rule is difficult to derive manually. However, the neural network controller may not perform well for conditions that are very different from the training data.

## Conclusion

This project successfully models and simulates a DC motor speed-control system, implements a PID controller, and develops a Machine Learning-based neural network controller. Both controllers are compared using standard control metrics. The project demonstrates that Machine Learning can be applied to intelligent control problems, while also showing that classical PID control remains useful due to its simplicity, interpretability, and reliability.
